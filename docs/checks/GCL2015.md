# GCL2015 — go-panic

> A function passed to wg.Go() may panic and bring the program down.

|           |                              |
|-----------|------------------------------|
| Code      | `GCL2015` |
| Slug      | `go-panic` |
| Primitive | `sync.WaitGroup` |

## Why it matters

An unrecovered panic in a Go() worker propagates and crashes the whole program.

## Examples

The linter flags code like this:

```go
wg.Go(func() {
	mustParse(input) // an unrecovered panic crashes the program
})
```

Write it like this instead:

```go
wg.Go(func() {
	if err := parse(input); err != nil {
		log.Print(err) // handle the error instead of panicking
	}
})
```

## Suppressing this check

Add an inline directive on the offending line. Either the canonical code or the legacy slug is accepted:

```go
foo() // goconcurrencylint:ignore GCL2015
foo() // goconcurrencylint:ignore go-panic
```
