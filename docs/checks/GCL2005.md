# GCL2005 — add-inside-goroutine

> wg.Add() is called from inside a worker goroutine, racing with Wait().

|           |                              |
|-----------|------------------------------|
| Code      | `GCL2005` |
| Slug      | `add-inside-goroutine` |
| Primitive | `sync.WaitGroup` |

## Why it matters

Wait() may observe a zero counter and return before the work is even registered.

## Examples

The linter flags code like this:

```go
for _, t := range tasks {
	go func(t Task) {
		wg.Add(1) // races with Wait(): may run after it returns
		defer wg.Done()
		t.Run()
	}(t)
}
wg.Wait()
```

Write it like this instead:

```go
for _, t := range tasks {
	wg.Add(1)
	go func(t Task) {
		defer wg.Done()
		t.Run()
	}(t)
}
wg.Wait()
```

## Suppressing this check

Add an inline directive on the offending line. Either the canonical code or the legacy slug is accepted:

```go
foo() // goconcurrencylint:ignore GCL2005
foo() // goconcurrencylint:ignore add-inside-goroutine
```
