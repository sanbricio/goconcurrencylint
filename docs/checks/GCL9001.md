# GCL9001 — sync-primitive-copy

> A sync primitive (or a struct embedding one) is copied by value.

|           |                              |
|-----------|------------------------------|
| Code      | `GCL9001` |
| Slug      | `sync-primitive-copy` |
| Primitive | `sync.Mutex`, `sync.RWMutex`, `sync.WaitGroup`, `sync.Once`, `sync.Cond`, `sync.Pool`, `sync.Map` |

## Why it matters

Copying duplicates the internal state, so the copy and the original stop synchronising — causing races, lost wakeups or panics.

## Examples

The linter flags code like this:

```go
type Counter struct {
	mu sync.Mutex
	n  int
}

func (c Counter) Inc() { // value receiver copies the mutex
	c.mu.Lock()
	defer c.mu.Unlock()
	c.n++
}
```

Write it like this instead:

```go
type Counter struct {
	mu sync.Mutex
	n  int
}

func (c *Counter) Inc() { // pointer receiver: no copy
	c.mu.Lock()
	defer c.mu.Unlock()
	c.n++
}
```

## Suppressing this check

Add an inline directive on the offending line. Either the canonical code or the legacy slug is accepted:

```go
foo() // goconcurrencylint:ignore GCL9001
foo() // goconcurrencylint:ignore sync-primitive-copy
```
