# GCL3001 — once-do-deadlock

> once.Do(f) where f calls Do on the same Once again — Once.Do is not reentrant, so this deadlocks.

|           |                              |
|-----------|------------------------------|
| Code      | `GCL3001` |
| Slug      | `once-do-deadlock` |
| Primitive | `sync.Once` |

## Why it matters

The inner Do waits for the outer Do to finish, which is waiting on the inner one — a deadlock.

## Examples

The linter flags code like this:

```go
var once sync.Once
once.Do(func() {
	once.Do(func() {}) // re-entrant Do on the same Once deadlocks
})
```

Write it like this instead:

```go
var once sync.Once
once.Do(func() {
	initialize() // no re-entrant Do on the same Once
})
```

## Suppressing this check

Add an inline directive on the offending line. Either the canonical code or the legacy slug is accepted:

foo() // goconcurrencylint:ignore once-do-deadlock
