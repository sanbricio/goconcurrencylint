# GCL1002 — unlock-without-lock

> An Unlock()/RUnlock() call is reached without a prior matching lock (including double-unlocks).

|           |                              |
|-----------|------------------------------|
| Code      | `GCL1002` |
| Slug      | `unlock-without-lock` |
| Primitive | `sync.Mutex`, `sync.RWMutex` |

## Why it matters

Unlocking a mutex that is not held panics at runtime ("sync: unlock of unlocked mutex").

## Examples

The linter flags code like this:

```go
func release(mu *sync.Mutex) {
	mu.Unlock() // mu was never locked in this scope
}
```

Write it like this instead:

```go
func critical(mu *sync.Mutex) {
	mu.Lock()
	defer mu.Unlock()
	// ... critical section
}
```

## Suppressing this check

Add an inline directive on the offending line. Either the canonical code or the legacy slug is accepted:

```go
foo() // goconcurrencylint:ignore GCL1002
foo() // goconcurrencylint:ignore unlock-without-lock
```
