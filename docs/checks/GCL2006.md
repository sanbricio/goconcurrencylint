# GCL2006 — done-not-deferred

> A worker calls Done() after an explicit panic or runtime.Goexit path instead of deferring it.

|           |                              |
|-----------|------------------------------|
| Code      | `GCL2006` |
| Slug      | `done-not-deferred` |
| Primitive | `sync.WaitGroup` |

## Why it matters

On the panic/Goexit path the Done() is skipped, so the counter never reaches zero and Wait() blocks.

## Examples

The linter flags code like this:

```go
go func() {
	if failed {
		panic("boom") // skips the Done() below
	}
	wg.Done()
}()
```

Write it like this instead:

```go
go func() {
	defer wg.Done() // runs even if the goroutine panics
	if failed {
		panic("boom")
	}
}()
```

## Suppressing this check

Add an inline directive on the offending line. Either the canonical code or the legacy slug is accepted:

```go
foo() // goconcurrencylint:ignore GCL2006
foo() // goconcurrencylint:ignore done-not-deferred
```
