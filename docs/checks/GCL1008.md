# GCL1008 — rwmutex-api-mismatch

> Unlock() is used for a read lock, or RUnlock() is used for a write lock.

|           |                              |
|-----------|------------------------------|
| Code      | `GCL1008` |
| Slug      | `rwmutex-api-mismatch` |
| Primitive | `sync.RWMutex` |

## Why it matters

Mismatched lock/unlock pairs corrupt the RWMutex state and panic or deadlock at runtime.

## Examples

The linter flags code like this:

```go
var mu sync.RWMutex
mu.RLock()
defer mu.Unlock() // RLock paired with Unlock instead of RUnlock
```

Write it like this instead:

```go
var mu sync.RWMutex
mu.RLock()
defer mu.RUnlock()
```
