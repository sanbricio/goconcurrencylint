# GCL2002 — done-without-add

> wg.Done() is called more times than wg.Add() allows, which panics at runtime.

|           |                              |
|-----------|------------------------------|
| Code      | `GCL2002` |
| Slug      | `done-without-add` |
| Primitive | `sync.WaitGroup` |

## Why it matters

Driving the counter below zero panics with "sync: negative WaitGroup counter".

## Examples

The linter flags code like this:

```go
var wg sync.WaitGroup
wg.Add(1)
wg.Done()
wg.Done() // drives the counter negative: panics
```

Write it like this instead:

```go
var wg sync.WaitGroup
wg.Add(1)
wg.Done()
```

## Suppressing this check

Add an inline directive on the offending line. Either the canonical code or the legacy slug is accepted:

```go
foo() // goconcurrencylint:ignore GCL2002
foo() // goconcurrencylint:ignore done-without-add
```
