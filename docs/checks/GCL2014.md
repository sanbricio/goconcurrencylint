# GCL2014 — done-outside-goroutine

> Done() runs on the parent goroutine instead of the worker, so a panic in the parent skips it.

|           |                              |
|-----------|------------------------------|
| Code      | `GCL2014` |
| Slug      | `done-outside-goroutine` |
| Primitive | `sync.WaitGroup` |

## Why it matters

If the parent panics before its Done(), the counter never reaches zero and Wait() blocks forever.

## Examples

The linter flags code like this:

```go
wg.Add(1)
go work()  // the worker never signals completion
wg.Done()  // the parent signals instead: Wait() may return early
wg.Wait()
```

Write it like this instead:

```go
wg.Add(1)
go func() {
	defer wg.Done() // the worker signals when it really finishes
	work()
}()
wg.Wait()
```

## Suppressing this check

Add an inline directive on the offending line. Either the canonical code or the legacy slug is accepted:

```go

foo() // goconcurrencylint:ignore done-outside-goroutine
```
