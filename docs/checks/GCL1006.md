# GCL1006 — mutex-in-loop

> A mutex is declared inside a loop body, creating a fresh lock per iteration.

|           |                              |
|-----------|------------------------------|
| Code      | `GCL1006` |
| Slug      | `mutex-in-loop` |
| Primitive | `sync.Mutex`, `sync.RWMutex` |

## Why it matters

A new mutex per iteration guards nothing shared, so the intended mutual exclusion never actually happens.

## Examples

The linter flags code like this:

```go
for i := range items {
	var mu sync.Mutex // a fresh lock each iteration guards nothing
	mu.Lock()
	process(items[i])
	mu.Unlock()
}
```

Write it like this instead:

```go
var mu sync.Mutex
for i := range items {
	mu.Lock()
	process(items[i])
	mu.Unlock()
}
```

## Suppressing this check

Add an inline directive on the offending line. Either the canonical code or the legacy slug is accepted:

```go
foo() // goconcurrencylint:ignore GCL1006
foo() // goconcurrencylint:ignore mutex-in-loop
```
