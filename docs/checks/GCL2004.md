# GCL2004 — go-after-wait

> wg.Go() is called after wg.Wait() returned empty — the Go 1.25 variant of add-after-wait.

|           |                              |
|-----------|------------------------------|
| Code      | `GCL2004` |
| Slug      | `go-after-wait` |
| Primitive | `sync.WaitGroup` |

## Why it matters

Reusing a drained group with Go() races with the prior Wait() the same way add-after-wait does.

## Examples

The linter flags code like this:

```go
wg.Wait()
wg.Go(worker) // reusing a drained group races with the prior Wait()
```

Write it like this instead:

```go
wg.Go(worker)
wg.Wait()
```

## Suppressing this check

Add an inline directive on the offending line. Either the canonical code or the legacy slug is accepted:

```go

foo() // goconcurrencylint:ignore go-after-wait
```
