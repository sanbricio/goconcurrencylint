# GCL1004 — unchecked-trylock

> TryLock()/TryRLock() is called without checking the returned boolean.

|           |                              |
|-----------|------------------------------|
| Code      | `GCL1004` |
| Slug      | `unchecked-trylock` |
| Primitive | `sync.Mutex`, `sync.RWMutex` |

## Why it matters

The code enters the critical section whether or not it actually acquired the lock, defeating the mutual exclusion.

## Examples

The linter flags code like this:

```go
func try(mu *sync.Mutex) {
	mu.TryLock() // result ignored: the lock may not be held
	defer mu.Unlock()
	// ... critical section
}
```

Write it like this instead:

```go
func try(mu *sync.Mutex) {
	if mu.TryLock() {
		defer mu.Unlock()
		// ... critical section
	}
}
```

## Suppressing this check

Add an inline directive on the offending line. Either the canonical code or the legacy slug is accepted:

