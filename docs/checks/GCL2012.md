# GCL2012 — multiple-done-worker

> The same worker branch can call Done() more than once.

|           |                              |
|-----------|------------------------------|
| Code      | `GCL2012` |
| Slug      | `multiple-done-worker` |
| Primitive | `sync.WaitGroup` |

## Why it matters

The extra Done() over-decrements the counter and panics.

## Examples

The linter flags code like this:

```go
go func() {
	defer wg.Done()
	if err != nil {
		wg.Done() // second Done() on this branch over-decrements
		return
	}
}()
```

Write it like this instead:

```go
go func() {
	defer wg.Done()
	if err != nil {
		return
	}
}()
```

## Suppressing this check

Add an inline directive on the offending line. Either the canonical code or the legacy slug is accepted:

```go
foo() // goconcurrencylint:ignore GCL2012
foo() // goconcurrencylint:ignore multiple-done-worker
```
