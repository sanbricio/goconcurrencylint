# GCL1003 — defer-unlock-without-lock

> A deferred Unlock()/RUnlock() can run while the mutex is unlocked.

|           |                              |
|-----------|------------------------------|
| Code      | `GCL1003` |
| Slug      | `defer-unlock-without-lock` |
| Primitive | `sync.Mutex`, `sync.RWMutex` |

## Why it matters

If the deferred unlock runs without a matching lock — or after a return/panic between defer and lock — it panics.

## Examples

The linter flags code like this:

```go
func work(mu *sync.Mutex) {
	defer mu.Unlock() // deferred before the lock is taken
	mu.Lock()
}
```

Write it like this instead:

```go
func work(mu *sync.Mutex) {
	mu.Lock()
	defer mu.Unlock()
}
```

## Suppressing this check

Add an inline directive on the offending line. Either the canonical code or the legacy slug is accepted:

