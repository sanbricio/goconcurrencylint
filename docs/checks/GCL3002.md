# GCL3002 — once-do-nil

> once.Do(nil) panics when the function is invoked.

|           |                              |
|-----------|------------------------------|
| Code      | `GCL3002` |
| Slug      | `once-do-nil` |
| Primitive | `sync.Once` |

## Why it matters

The first run dereferences a nil function value and panics.

## Examples

The linter flags code like this:

```go
var once sync.Once
once.Do(nil) // invoking a nil function value panics
```

Write it like this instead:

```go
var once sync.Once
once.Do(func() {
	initialize()
})
```

## Suppressing this check

Add an inline directive on the offending line. Either the canonical code or the legacy slug is accepted:

```go

foo() // goconcurrencylint:ignore once-do-nil
```
